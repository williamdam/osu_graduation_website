JS + DOM + Cloud Firestore = Realtime Updates

This is a demonstration of Javascript and interaction with a NoSQL database. I've setup a Javascript listener to get user input, then stores that user input with a timestamp on a Firestore Database. There's a snapshot listener, which checks for changes in metadata on the database, then updates the DOM, accordingly. The data read from the database is sorted by timestamp, in ascending order, and auto scrolls to the bottom.

Note: Database connection doesn't use authentication, for demonstration purposes. An actual deployment would involve user authentication and the user IDs displayed on each post.

Technical Points: HTML, CSS, Javascript, DOM Manipulation, NoSQL (Firestore DB)

Lessons Learned: Database interaction and realtime updates without refreshing the browser.
